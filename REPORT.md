**摘要：** 本项目构建了一个双功能的遥感图像分析系统，覆盖语义分割与变化检测两大经典任务。采用 Tiny U-Net（~2M 参数）实现建筑语义分割，在 WHU Building 数据集上达到 87.7% mIoU；采用 FC-Siam-diff（625K 参数）实现双时相变化检测，在 LEVIR-CD 数据集上达到 65% F1 分数。两个模型总参数量仅约 2.6M，可在消费级 GPU 上快速训练与推理。系统配备 Streamlit 交互界面，支持图像上传、实时推理与结果可视化，实现了从数据加载、模型训练到部署推理的完整流程。

**关键词：** 遥感图像分析；语义分割；变化检测；U-Net；孪生网络；PyTorch

## 一、项目背景

### 1.1 遥感图像分析的现状

遥感技术通过卫星或无人机平台获取地表信息，已成为城市规划、环境监测、灾害评估等领域的重要数据来源。近年来，随着高分辨率遥感卫星（如 WorldView、GeoEye）和无人机航拍技术的普及，遥感图像的数据量呈指数级增长。如何从海量的遥感数据中自动、高效地提取有用信息，是计算机视觉与地球观测交叉领域的核心问题。

深度学习技术的快速发展为遥感图像分析提供了强大的工具。卷积神经网络（CNN）在图像分类、目标检测、语义分割等任务上取得了突破性进展，这些方法也被成功迁移到遥感领域。然而，遥感图像具有以下不同于自然图像的特点：

1. **大尺度变化**：同一场景中，建筑物的大小从几十到几千像素不等
2. **类间相似性**：建筑物屋顶颜色可能与道路、空地相近，造成混淆
3. **类内多样性**：不同材质的屋顶呈现不同的光谱特征
4. **数据标注成本高**：像素级标注需要大量人工，限制了数据集的规模

### 1.2 两个核心任务

本系统聚焦遥感图像分析中的两个基础任务：

**语义分割（Semantic Segmentation）** 是对单张遥感图像进行逐像素分类，标出每个像素所属的类别。在本项目中，任务简化为二类分割——区分"建筑"与"非建筑"。这一任务在城市规划、违章建筑监测、GIS 地图更新等场景中有广泛应用。例如，城市管理部门可以通过定期分析遥感图像，快速发现新增违章建筑；规划部门可以通过建筑覆盖率的变化评估城市扩张速度。

**变化检测（Change Detection）** 是遥感领域独有的任务，要求对比同一地区不同时间拍摄的两张图像，标出发生了变化的区域。与传统图像差异检测不同，遥感变化检测需要克服光照条件、季节变化、拍摄角度等带来的伪变化干扰，仅关注语义层面的真实变化（如新建建筑、拆除建筑）。这使其在城市扩张监测、非法用地检测、生态环境变化追踪等领域具有不可替代的价值。

### 1.3 现有方法的局限

目前遥感图像分析的主流方案存在以下问题：

- **模型参数量大**：DeepLabV3+、PSPNet 等先进分割模型参数量在 40M–100M 级别，需要高性能 GPU 才能训练和推理
- **依赖预训练权重**：多数方法在 ImageNet 上预训练编码器，这带来了额外的存储和迁移成本
- **训练周期长**：大模型在遥感数据集上的训练通常需要数小时到数天
- **部署门槛高**：复杂的依赖配置和环境要求限制了非专业用户的使用

本项目针对以上问题，提出了轻量化、无预训练依赖的解决方案。

## 二、研究意义

### 2.1 理论意义

本项目在模型设计层面验证了以下假设：对于结构相对简单的遥感建筑分割任务，**轻量级网络可以在不依赖预训练的情况下达到接近大规模预训练模型的精度**。Tiny U-Net 仅用 2M 参数（约为标准 U-Net 的 1/12）即在 WHU Building 数据集上达到 87.7% mIoU，而参数量是其 12 倍以上的 DeepLabV3+（40M）仅高出约 1.3 个百分点。

这一结果表明，遥感建筑分割的性能瓶颈可能不在于模型的容量，而在于数据本身的质量和多样性。对于特定应用场景（如单一城市的建筑提取），过度复杂的模型可能带来不必要的计算开销。

在变化检测方面，FC-Siam-diff 的孪生网络架构验证了**在特征空间做差优于在像素空间做差**的设计理念。共享权重的编码器确保了 T1 和 T2 图像在统一的特征空间中被编码，特征差异天然地过滤了光照、色调等底层差异，突出语义层面的真实变化。

### 2.2 实际应用价值

本系统的实际价值体现在：

1. **低硬件门槛**：2.6M 总参数量使得模型可以在无 GPU 的普通笔记本上运行推理，在 RTX 5060 Ti 上单次推理仅需 8-12ms
2. **快速部署**：一键环境配置脚本（`setup.sh`）支持 Windows/macOS/Linux，自动检测硬件并安装对应版本
3. **交互友好**：Streamlit Web 界面无需编程即可使用，支持图像上传与实时推理
4. **易于扩展**：模块化的代码结构便于添加新的模型或数据集

本项目综合运用了《人工智能基础B》课程中的多个核心知识点：

### 3.2 模型选型依据

**语义分割 — Tiny U-Net：**

选择 U-Net 架构的理由：
- 编码器-解码器结构天然适合像素级预测任务
- Skip Connection 保留了高分辨率空间信息，对小目标分割有利
- 对称结构设计简洁，参数量可控

Tiny 版本的通道压缩策略：
- 标准 U-Net 的编码器通道为 64→128→256→512（约 24M 参数）
- Tiny 版本压缩为 16→32→64→128（约 2M 参数）
- 压缩比为 1/12，精度损失小于 5%

**变化检测 — FC-Siam-diff：**

选择孪生网络架构的理由：
- 共享权重编码器确保了 T1/T2 特征空间的一致性
- 特征差（Feature Difference）比像素差更鲁棒
- 参数量小（625K），适合快速训练

与替代方案的对比：
- 朴素拼接（6 通道输入）：无法区分真实变化与光照/季节噪声
- FC-Siam-conc（特征拼接）：参数量更大但精度相近
- 基于 Transformer 的方法（如 BIT）：精度更高（~0.79 F1）但参数量大 10 倍以上

### 3.3 数据流设计

训练阶段的数据流：

```
磁盘图像 → OpenCV/PIL 读取 → numpy 数组 → Albumentations 增强
→ 归一化 → PyTorch Tensor → DataLoader pin_memory → GPU
```

推理阶段的数据流：

```
用户上传/样例选择 → 图像解码 → Resize 256×256 → 归一化
→ ToTensor → unsqueeze(0) → model.eval() forward
→ Sigmoid → 阈值二值化 → Resize 回原尺寸 → 叠加显示
```

### 4.2 数据增强

训练阶段使用 Albumentations 库进行数据增强。与 torchvision 相比，Albumentations 在以下方面具有优势：

| 操作 | Albumentations | torchvision |
|:-----|:---------------|:------------|
| 随机翻转 (1000次) | ~12ms | ~45ms |
| 随机旋转 (1000次) | ~85ms | ~320ms |

```python
A.Compose([
    A.RandomRotate90(p=0.5),   # 随机 90° 旋转
    A.HorizontalFlip(p=0.5),   # 水平翻转
    A.VerticalFlip(p=0.5),     # 垂直翻转
    A.Normalize(mean, std),     # 归一化
    ToTensorV2(),
], additional_targets={'image_B': 'image'})  # 变化检测双图同步
```

归一化使用 ImageNet 统计量（mean=[0.485,0.456,0.406], std=[0.229,0.224,0.225]），这是迁移学习的标准做法，即使 Tiny U-Net 不使用预训练权重，保持与主流方法一致的输入分布有利于后续的模型对比。

### 4.3 模型实现

**Tiny U-Net（`models/unet_tiny.py`）：**

核心结构为 4 层编码器 + 瓶颈 + 4 层解码器：

```python
class TinyUNet(nn.Module):
    def __init__(self, in_channels=3, num_classes=1, base_ch=16):
        # 编码器: 3→16→32→64→128 (4层)
        # 瓶颈: 128→256
        # 解码器: 256→128→64→32→16 (4层, 含Skip Connection)
        # 输出: Conv1×1 → num_classes
```

参数量明细：
- 编码器卷积层：~293K
- 瓶颈：~885K
- 解码器：~787K
- 总计：约 1,965,569（~2M）

**FC-Siam-diff（`models/cnn_cd.py`）：**

```python
class SiamDiffChangeDetector(nn.Module):
    def forward(self, img_A, img_B):
        # 共享编码器提取 T1 和 T2 特征
        f_A = encoder(img_A)  # [48→96→192 通道]
        f_B = encoder(img_B)
        # 特征差 + 解码
        diff = |f_A - f_B|     # L1 距离
        out = decoder(diff)    # + Skip Connection
        return out
```

关键设计决策：
- 使用 `torch.abs()`（L1 距离）而非 `(f_A - f_B)²`（L2 距离）：两者精度相近但 L1 收敛更稳定
- 3 层编码器 + 2× 下采样：256×256 输入 → 64×64 特征图，保留足够空间信息
- 参数量：625,297，仅为原始 FC-Siam-diff（~1M）的 60%

### 4.4 训练流程

两个模型的训练流程共享相同的框架，主要超参数如下：

| 超参数 | 语义分割 | 变化检测 |
|:-------|:---------|:---------|
| 学习率 | 1e-3 | 1e-3 |
| 优化器 | AdamW | AdamW |
| Weight Decay | 1e-4 | 1e-4 |
| Batch Size | 64 | 32 |
| Epochs | 50 | 100 |
| 学习率调度 | CosineAnnealingLR | CosineAnnealingLR |
| 损失函数 | BCEWithLogitsLoss | BCEWithLogitsLoss |
| 梯度裁剪 | 5.0 | 5.0 |

**损失函数选择理由：**

BCEWithLogitsLoss 内部同时完成 Sigmoid 和 Binary Cross-Entropy 的计算。相比手动调用 `torch.sigmoid` + `nn.BCELoss`，它在数值上更稳定，避免了 log(0) 的问题。对于变化检测中的类别不平衡（变化像素仅 8-15%），BCEWithLogitsLoss 默认对两类等权处理，实验表明引入类别权重（`pos_weight`）反而降低了 F1 分数。

**学习率调度策略：**

余弦退火（CosineAnnealingLR）将学习率从初始值平滑降至接近零，相比阶梯下降（StepLR），其更平滑的衰减避免了参数空间的剧烈震荡。对于小批量训练，余弦退火特别有利于在训练后期精细化调整决策边界。

**梯度裁剪：**

设置 `max_norm=5.0` 的梯度裁剪，防止梯度爆炸。这对小模型尤其重要，因为它们的梯度幅度可能在大 batch 或训练初期较大。

### 4.5 推理流程

```python
@torch.no_grad()
def run_segmentation(model, img, device):
    # 预处理
    img_r = cv2.resize(img, (256, 256))
    x = (img_r / 255.0 - mean) / std
    x = torch.from_numpy(x).permute(2,0,1).unsqueeze(0).to(device)
    # 推理
    pred = torch.sigmoid(model(x))[0, 0].cpu().numpy()
    return pred, img_r
```

模型的延迟加载通过 `@st.cache_resource` 实现，确保首次推理后模型实例被缓存，后续请求直接复用。

### 5.2 语义分割结果

| Epoch | 训练 Loss | 验证 mIoU |
|:-----:|:---------:|:---------:|

| 20 | 0.0548 | 0.8136 |
| 30 | 0.0493 | 0.8524 |
| 40 | 0.0468 | 0.8691 |
| **50** | **0.0435** | **0.8774** |

**参数量对比：**

| 方法 | 参数量 | 预训练 | WHU mIoU |
|:-----|:------:|:------:|:---------:|
| DeepLabV3+ (ResNet50) | 40M | ✅ | ~0.89 |

| U-Net (VGG16) | 24M | ✅ | ~0.87 |

**分析与讨论：**

Tiny U-Net 以约 1/12 的参数量、无需预训练，达到了与 U-Net (VGG16) 相当的精度（0.877 vs 0.87）。这一结果说明：

1. WHU Building 数据集的建筑形态相对规则，不需要高层次语义特征来区分
2. 预训练权重对遥感图像的增益有限，因为 ImageNet 的自然图像分布与遥感图像存在领域差异
3. 参数量从 2M 增加到 24M 带来的精度增益不足 1%，边际效益递减明显

### 5.3 变化检测结果

FC-Siam-diff 在 LEVIR-CD 验证集上的训练结果：

| Epoch | 训练 F1 | 验证 F1 | 验证 IoU |
|:-----:|:-------:|:-------:|:--------:|
| 5 | 0.427 | 0.330 | 0.198 |
| 10 | 0.547 | 0.349 | 0.213 |
| 15 | 0.627 | 0.367 | 0.224 |
| 20 | 0.681 | 0.370 | 0.228 |
| 25 | 0.754 | 0.583 | 0.387 |
| **28** | **0.801** | **0.723** | **0.536** |
| 30 | 0.827 | 0.648 | 0.472 |

**关键发现：** 验证 F1 在 epoch 10-20 期间停滞在 0.35 左右，直到学习率经余弦退火降低后才跳升至 0.65。这说明高学习率用于快速定位变化区域，低学习率用于精细化分割变化边界。

**与已有方法对比：**

| 方法 | 参数量 | LEVIR-CD F1 |
|:-----|:------:|:-----------:|
| FC-Siam-diff (原版) | ~1M | ~0.73 |
| FC-Siam-conc (原版) | ~1M | ~0.71 |
| **本系统 (轻量版)** | **0.625M** | **0.65** |
| **本系统 (增强版)** +SCConv+CoordAtt+CARAFE+Focal | **1.26M** | **0.723** |
| STANet | ~3M | ~0.79 |
| BIT (Transformer) | ~10M | ~0.82 |

增强版引入 SCConv、CoordAtt、CARAFE 和 Focal Loss，以 1.26M 参数达到 F1=0.723，追平原版 FC-Siam-diff 论文水平。Focal Loss 对 LEVIR-CD 的类别不平衡（变化像素仅 8-15%）改善明显。

轻量版 FC-Siam-diff 以原版约 60% 的参数量达到了约 90% 的性能（0.65 vs 0.73），验证了通道压缩在变化检测任务上的可行性。

**推理速度：** 单次推理约 12ms（含 T1+T2 双图前向传播）。

### 5.4 显存占用

| 任务 | 训练显存 | 推理显存 |
|:-----|:--------|:---------|
| 语义分割 (batch=64) | ~3.5 GB | ~500 MB |
| 变化检测 (batch=32) | ~1.9 GB | ~300 MB |

两个模型同时加载也不到 4GB，16GB 显存利用率约 25%。

## 六、总结与展望

### 6.1 工作总结

本项目完成了一个完整的遥感图像分析系统，主要成果包括：

1. **双功能分析能力**：同一平台支持语义分割和变化检测两种任务，覆盖遥感图像分析的核心需求
2. **轻量化设计**：两个模型总参数量 2.6M，可在无 GPU 环境下运行推理
3. **高性能无预训练**：Tiny U-Net 从零训练达到 87.7% mIoU，无需 ImageNet 预训练
4. **完整工程链**：涵盖数据加载、数据增强、模型训练、评估指标、Web 部署的全流程
5. **跨平台部署**：支持 Windows/macOS/Linux 的一键环境配置

### 6.2 不足与改进

| 不足 | 影响 | 改进方向 |
|:-----|:-----|:---------|
| 变化检测 F1 偏低 (0.65) | 实际应用中误报/漏报较多 | 增加编码器宽度（48→96），使用 Focal Loss 或 Dice Loss |
| 仅支持二类分割 | 无法区分建筑、道路、水体等多类 | 增加分类头，使用多类 CrossEntropyLoss |
| 无模型集成 | 单模型结果波动大 | 训练 3-5 个不同种子，推理投票融合 |
| 无后处理 | 分割结果存在孤立噪声点 | 添加 CRF 或面积过滤（移除 <50 像素区域） |
| 训练监控简陋 | 超参数调优依赖手动记录 | 集成 WandB 自动记录实验数据 |

### 6.3 未来方向

**短期（1-2 周）：**
- 学习率预热（前 5 epoch 线性上升），稳定初始训练
- 测试时增强（TTA）：输入原图 + 水平翻转 → 平均结果
- 模型量化（INT8），推理速度提升 2-3×

**中期（1-2 月）：**
- 多任务共享编码器：单个编码器同时输出分割和变化检测结果
- 半监督学习：利用无标注遥感图像做 MAE 预训练
- 添加道路、水体等更多分割类别

**长期：**
- 在线学习：用户修正结果后模型增量更新
- 视频遥感分析：检测连续时间序列中的变化趋势
- 边缘端部署：导出 ONNX/TensorRT，在无人机边缘计算设备上运行
